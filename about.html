About

This tool is still in development and should not be considered a finished product.

Purpose

Plotter is a tool that creates scatterplots of NFL player data. It was developed specifically with fantasy football in mind, to help visualize trends that occur over time at a positional level. For instance, graphing fantasy points against position rank for 2012 and 2008 shows that in general the n-th highest scoring running back contributed substantially more points over a season in 2008 than in 2012. These kinds of insights can–and should–inform draft strategy and roster construction.

Design

I based the visual style on a set of flat form elements I had previously developed. Colors are cribbed from Flat UI Colors. The typeface used here is Effra, served by Typekit.

Code

Plotter runs on Node.js and is currently hosted on OpenShift. It was developed in the browser with Cloud9 IDE. The Node server is extremely light and doesn't connect to a database. The dataset is a single large JSON file, delivered to the client asynchronously, and all code runs on the client as well. The script has been rewritten a few times; currently it uses the revealing module pattern and is compiled from CoffeeScript.

The chart drawing function uses d3.js, a great data visualization library. Other libraries include: Color.js for color manipulation; Chosen.js with a custom stylesheet using relative units and a tiny data-URI font for icons; and of course jQuery.

Data

This dataset is collected from the fantasy stats provided by PFR. Currently it contains the top 60 players per season at each of the four offensive fantasy football positions. I Plan to continue to update it annually.

Future

There are a number of tweaks, updates, and improvements I plan to make.

  • Add a slider that will set the limit on the position rank of the players returned. For instance, a user could view only the top 20 running backs rather than getting all 60.
  • Retain the JSON dataset with localStorage, and add some kind of version detection to load and save the dataset from the server when it's updated.
  • Improve input validation and provide contextual help when users fail to complete the fields required to produce a scatterplot.
